📖 内容综述

Transformer是所有现代大语言模型（GPT、Claude、LLaMA）的底层引擎。这份笔记把它从头拆到尾：先从语言模型的进化史讲起（词袋→词嵌入→注意力→Transformer），让你明白为什么Transformer能成为"唯一的王"；然后逐步拆解六个核心组件——分词、嵌入、位置编码、自注意力、多头注意力、前馈网络；最后讲文本生成的自回归过程和最新的优化技术（KV Cache、MoE）。

理解Transformer不需要数学博士学位——每个公式都会配上白话解释和具体例子，复杂的架构会先拆成小块理解，最后再串成完整全貌。

一、语言模型进化史：为什么需要Transformer

📐 Transformer完整架构总览

📐 语言模型进化四步: 从词袋到Transformer

从词袋到Transformer的四步跳跃

阶段 技术 核心思想 致命缺陷
词袋模型 Bag of Words 统计每个词出现的次数 "我爱你"和"你爱我"完全一样——丢失了顺序
词嵌入 Word2Vec 用向量表示词，语义相近的词距离近 一个词永远只有一个向量——"苹果"(水果)和"苹果"(公司)分不开
RNN+注意力 Seq2Seq+Attention 按顺序处理+关注重要部分 必须串行计算——像排队过独木桥，慢得要命
Transformer Self-Attention 所有位置并行处理，每个词直接看到所有其他词 ✅ 无明显缺陷，成为"唯一的王"

💡 关键认知：Transformer的革命性在于一个字——并。RNN是一个词一个词排队处理（串行），Transformer是所有词同时处理（并行）。这就是为什么Transformer能训练千亿参数模型，而RNN做不到。

二、分词（Tokenization）：拆乐高积木

一句话：分词就是把人类可读的文字切成模型能处理的"碎片"（Token），像把一堆乐高积木拆成单个零件。

📐 输入处理三步: 分词→嵌入→位置编码

三种分词策略

方式 例子 优劣
按词 ["This", "is", "a", "cat"] 语义清晰，但词表巨大
按字符 ["T","h","i","s",...] 词表小，但丢失语义
子词(BPE) ✅ ["un", "happi", "ness"] 平衡词法和语义，主流选择

现代LLM（GPT/LLaMA）几乎都用BPE（字节对编码）——它能把"unhappiness"拆成有意义的子词，既保留语义又控制词表大小。

三、嵌入（Embedding）：给积木块赋予特征

一句话：嵌入就是把离散的Token ID变成一个高维向量——让"苹果"和"梨"因为都有"水果"特征而在数学空间里靠近。

Token: "apple"
  ↓ 查嵌入表
向量: [0.99, -0.05, 0.23, ..., -0.12]  (300-12288维)

整个句子变成一个矩阵：

"This is a cute cat" → 分词 → 查嵌入表 →

[0.12, -0.45, ...]  ← This
[-0.67, 0.12, ...]  ← is
[0.89, -0.01, ...]  ← a
[0.23, -0.89, ...]  ← cute
[0.56, -0.76, ...]  ← cat

→ 矩阵形状: (5, d_model)

💡 关键认知：嵌入不是随机初始化后不变的——它在训练过程中不断更新。训练结束后，语义相似的词（如"国王"和"女王"）在向量空间中真的很近，这就是Word2Vec发现的"king - man + woman = queen"现象。

📐 Embedding: 语义空间中的词向量 — 含义相近的词距离更近

四、位置编码（Positional Encoding）：给积木贴身份证

一句话：Transformer并行处理所有Token，导致"我吃苹果"和"苹果吃我"看起来一样——必须手动注入位置信息来区分顺序。

📐 位置编码: 为什么Transformer需要"身份证"

为什么需要位置编码？

自注意力机制是"置换不变"的——它不关心Token的前后顺序。但语言严重依赖顺序！所以必须给每个Token贴一张"身份证"标明它是第几个。

正弦位置编码的设计

PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

白话解释：像收音机的多频道设计—— - 低维度（低频）：变化慢，捕捉长距离位置差（第1词 vs 第100词） - 高维度（高频）：变化快，捕捉短距离位置差（相邻两词） - 每个位置的"频率组合"像指纹一样独一无二

核心优势：支持相对位置学习——模型学会"隔两个词"的关系后，能自动泛化到任意位置。

五、自注意力（Self-Attention）：图书馆检索系统

一句话：自注意力让序列中的每个词都能"看到"所有其他词，并根据相关性自动分配权重。

📐 自注意力 Q/K/V 计算流程

📐 Self-Attention: Q·K·V 三步计算

Q、K、V三剑客（图书馆比喻）

角色 全称 比喻 作用
Q (Query) 查询向量 你手中的搜索关键词 "我要找什么？"
K (Key) 键向量 每本书的标题/索引 "我是什么？"
V (Value) 值向量 书的具体内容 "我有什么信息？"

计算过程

Attention(Q, K, V) = softmax(QK^T / √d_k) × V

白话四步： 1. 用你的Q去和所有书的K"对暗号"（点积计算相似度） 2. 除以√d_k防止数值太大（缩放因子） 3. Softmax把分数变成概率（权重） 4. 按权重从V中加权提取信息

具体例子："我爱学习AI"

目标词：「爱」
对各词的关注度：
  我：0.1   ← 主语，有关联
  爱：0.2   ← 自己
  学习：0.4  ← 主要对象，高权重 ⭐
  AI：0.3   ← 学习的内容，有关联

结果 = 0.1×"我" + 0.2×"爱" + 0.4×"学习" + 0.3×"AI"
     = 「爱」在这句话中的上下文表示

💡 关键认知：自注意力的精髓是——每个词不再是孤立的，而是带着整句话的语境。"苹果"出现在"吃苹果"里和"苹果公司"里，得到的向量是完全不同的——因为上下文中其他词的影响不同。

六、多头注意力（Multi-Head Attention）：多个观察员

一句话：单头注意力只能捕捉一种关系模式。多头让模型同时从不同"视角"观察信息——一个头看语法、一个头看指代、一个头看长距离依赖。

📐 多头注意力: 多视角并行观察

为什么需要多头？

Head 1：关注语法关系（主谓宾）
Head 2：关注指代关系（"他"指谁）
Head 3：关注短语搭配（"机器学习"是一个整体）
Head 4：关注长距离依赖（句首和句尾的关系）
...
多个头的输出拼接 → 融合层 → 丰富的综合表示

计算流程

输入 X (seq_len, 512)
  ↓
分成8个头，每头维度=512/8=64
  ├─ Head1: Attention(Q1,K1,V1) → (seq_len, 64)
  ├─ Head2: Attention(Q2,K2,V2) → (seq_len, 64)
  ├─ ...
  └─ Head8: Attention(Q8,K8,V8) → (seq_len, 64)
  ↓
拼接所有头 → (seq_len, 512)
  ↓
线性融合层 W^O → (seq_len, 512)

💡 关键认知：多头注意力不增加总计算量（每个头处理512/8=64维），但大幅提升表达能力。这就像8个人同时从不同角度看同一个问题，比1个人看得全面得多。

七、Transformer块：组装完整引擎

每一层Transformer由两个子模块组成，堆叠N层（6到96层不等）：

输入 X
  ↓
┌─────────────────────────┐
│  多头注意力               │
│  + 残差连接 (X + Attn(X)) │
│  + 层归一化 (LayerNorm)   │
└─────────────────────────┘
  ↓
┌─────────────────────────┐
│  前馈网络 (FFN)          │
│  Linear(升维512→2048)    │
│  ReLU激活                │
│  Linear(降维2048→512)    │
│  + 残差连接 + LayerNorm  │
└─────────────────────────┘
  ↓
输出（送入下一层）
组件 作用 比喻
多头注意力 捕捉词间关系 采购员：全球搜集最佳食材
前馈网络(FFN) 深度加工信息 大厨：将食材高温烹饪
残差连接 防梯度消失 保底通道：万一加工出错还有原料
LayerNorm 稳定训练 质检员：确保半成品合格

📐 Transformer块: 一层的完整计算流

💡 关键认知：Transformer的核心突破不是"性能更好"，而是"可以并行"。RNN必须一个词一个词地顺序处理（前一个算完才能算后一个），而Transformer通过Self-Attention让所有词同时互相关注，训练速度提升了几十倍。没有这个并行化，今天的GPT-4/Claude这种万亿参数的模型根本训练不出来。

八、文本生成：自回归的"逐字接龙"

一句话：模型用自己之前输出的词作为下一轮的输入——一个词一个词地"接龙"出完整句子。

人写文章的过程：
想好 → "我" → 看着"我"想 → "爱" → 看着"我爱"想 → "学习" → ...

模型也是这样：
<start> → 模型预测"我"
<start>我 → 模型预测"爱"
<start>我爱 → 模型预测"学习"
<start>我爱学习 → 模型预测"AI"
<start>我爱学习AI → 模型预测<end>

📐 自回归生成: 逐词预测过程

掩蔽自注意力（Causal Masking）

生成时模型只能看前面的词，不能"偷看"后面：

      我   爱   学习  AI
我    ✅   ❌   ❌   ❌
爱    ✅   ✅   ❌   ❌
学习  ✅   ✅   ✅   ❌
AI    ✅   ✅   ✅   ✅

下三角掩码保证了单向信息流——这就是GPT等"仅解码器"模型的核心机制。

💡 关键认知：自回归是"逐步构建上下文"的过程。每生成一个词，上下文就多了一个词，下一次预测就有更多信息可参考。这既是模型"越写越顺"的原因，也是生成速度慢的根源（必须一个个生成，无法并行）。

九、最新优化技术

技术 解决什么问题 核心思路
KV Cache 重复计算历史Token的K/V 缓存已计算的K/V，新Token只算自己的
Multi-Query Attention KV缓存占内存太大 多个Query头共享一组K/V
Grouped-Query Attention MQA质量损失 折中：每组几个头共享K/V
Sparse Attention 超长序列O(n²)太慢 只关注局部或稀疏位置
MoE（混合专家） 参数太多推理太慢 每次只激活部分"专家"网络

十、核心洞察

1. 并行计算是Transformer的命脉

自注意力让所有位置同时交互——这是它能训练千亿参数的根本原因。

2. 位置编码是"补丁"

Transformer天生不知道顺序，位置编码是后打的补丁。但正弦编码的巧妙设计让它能学会"相对位置"。

3. 多头=多视角

8个头 ≈ 8个不同角度的观察员，总计算量不变但信息量翻倍。

4. FFN是"知识存储"

注意力负责"关系建模"，FFN负责"深度加工"。研究表明，FFN层存储了大量事实性知识。

5. 自回归是双刃剑

逐词生成保证了质量（每步都有上下文），但也限制了速度（无法并行生成）。KV Cache等优化正是为了缓解这个问题。
