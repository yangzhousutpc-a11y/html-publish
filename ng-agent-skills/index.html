📖 内容综述

这门课回答的是一个很实际的问题：怎么让AI Agent从"每天失忆的实习生"变成"靠谱的老员工"？ 答案是Skill——把特定领域的最佳实践（品牌规范、法务流程、业务指标）打包成标准化文件夹，Agent加载后就能像专家一样工作。

课程分五个模块递进展开：先用四大设计模式（反思/工具/规划/多Agent）打好Agent的架构基础，再深入Skill的三层加载机制（元数据→指令→资源）解决Token经济学问题，最后讲MCP协议如何让工具标准化连接，以及评估与误差分析为什么才是Agent开发成败的真正关键。

一、四大Agent设计模式

📐 四大Agent设计模式总览

1. 反思模式（Reflection）

一句话：让AI先写初稿，再自我批评，然后根据批评修改——模拟人类"写→审→改"的过程。

生成初稿 → 自我反思/批评 → 根据反馈修正 → 最终输出

课上例子： - 让GPT写代码 → 运行测试 → 把错误反馈给GPT → GPT修复 - 效果：代码正确率从48%提升到74%（甚至超越未优化的GPT-4）

💡 关键认知：反思模式"相对容易实施，但性能提升令人惊讶"。只是多调用一次LLM让它"反省"，质量就能显著提升。

📐 反思模式(Reflection)流程

2. 工具使用（Tool Use）

一句话：让模型能调用外部函数——查天气、执行代码、访问数据库——从"只会说"变成"能做事"。

课上例子：两个Agent通过调用棋盘工具进行国际象棋对弈。

📐 工具使用(Tool Use)流程

3. 规划模式（Planning）

一句话：面对复杂任务，Agent先制定步骤计划，再逐步执行，遇到意外动态调整。

两种范式： - ReAct：走一步看一步（动态循环） - Plan-and-Execute：先画蓝图再施工（全局规划）

📐 规划模式 ReAct循环

4. 多Agent协作（Multi-Agent）

一句话：放弃造"通才"，组建"专才"团队——不同Agent扮演不同角色，分工协作完成复杂任务。

课上例子： - 脱口秀对话：两个脱口秀演员Agent进行双人喜剧 - 客户入职：多个Agent按流程做问卷、推荐产品、收集反馈 - 股票报告：数据分析员→金融分析师→风控员→投资经理

模式 核心机制 解决的问题 适用场景
反思 自我审查 减少错误 代码审查、文本润色
工具 调用外部API 知识和能力扩展 数据分析、实时查询
规划 任务拆解 复杂任务不迷失 项目管理、多步工作流
多Agent 角色分工 单一视角局限 团队协作、复杂决策

二、Agent Skill：从烟囱到乐高

📐 从烟囱模式到乐高模式: Agent能力组织方式的进化

旧模式 vs 新模式

维度 旧模式：烟囱式 新模式：通用Agent+Skill库
做法 每个场景造一个专用Agent 一个通用Agent + 按需加载Skill
问题 各自为战，无法复用 乐高式组合，一处维护到处用
维护 成本极高（软件反模式） 修改Skill自动影响所有Agent
例子 搜索Agent/编程Agent/营销Agent分开搞 Skill库：BigQuery+分析+品牌+PPT

💡 关键认知：从烟囱到乐高的转变，本质是从"每次重写"到"积累复用"。一个写过的"数据分析Skill"，下次遇到类似需求直接加载就行——Agent从零开始的时代结束了。这就是为什么Skill被称为"Agent时代的npm包"。

Skill到底是什么？

一句话：Skill是一个有组织的文件夹，内含指令、脚本和资源，让Agent能精准执行特定任务。

类比：Skill不是让AI变聪明，而是告诉AI"在你的公司，事情是怎么做的"。

📐 多Agent协作: 角色分工模式

📐 Skill三层渐进式披露

三层渐进式披露（Token经济学设计）

为什么要分层？ 因为Context Window有限、Token要钱。不可能把所有Skill信息一次性塞给Agent。

层级 内容 加载时机 Token消耗 类比
第1层：元数据 name + description 始终加载 极少 员工简历
第2层：指令 skill.md完整正文 选中后加载 中等 操作手册
第3层：资源 Reference文档 + Script脚本 按需加载 可选 参考附录

核心价值：解决了"数百个Skill与有限上下文窗口"的矛盾。

💡 关键认知：三层披露的精妙之处在于Token经济学——Context Window有限、Token要钱。如果把所有Skill的完整内容都塞进去，Token立刻爆炸。分层按需加载，让Agent用最少的Token做最精准的决策。这是工程化思维在AI领域的完美体现。

Skill的三个能力维度

维度 解决什么问题 例子
领域知识 不懂行业规则 品牌规范、法务流程
可复用工作流 重复任务无法标准化 每周复盘、季度回顾
新能力扩展 原本不具备的能力 生成PPT、构建MCP Server

💡 关键认知：Skill的本质不是"更好的Prompt"，而是"结构化的知识管理"。它让企业能把多年的业务经验、SOP流程标准化地编码进AI——这是从"聊天机器人"到"企业级AI助手"的跨越。

四、评估与误差分析：Agent开发的"道"

吴恩达反复强调：四大模式只是"术"，评估才是"道"。大部分团队失败的原因是"凭感觉优化"而非数据驱动。

📐 AI Agent 四大设计模式

📐 Agent评估循环: 数据驱动的迭代改进

评估循环

构建初始工作流 + 测试任务
        ↓
端到端评估（最终结果质量）
        ↓
组件级评估（每一步准确性）
        ↓
精准改进（只优化有问题的环节）
        ↓
    [循环] 直到达标

实战案例

发票日期错误： 1. 建10-20个测试发票（含易混淆日期） 2. 手动提取正确日期作为基准 3. 对比Agent提取结果，统计错误率（如20%） 4. 若超阈值（10%）→ 优化日期提取工具

💡 关键认知："一个简陋但自动化的评估器"远好于"强大但手工的评估流程"。哪怕只写一个5个测试用例的脚本，也比每次人肉检查强百倍。
