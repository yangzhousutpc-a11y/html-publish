房价预测的例子

课上用的第一个例子：给定房屋面积，预测房价。

面积 → [某个函数] → 房价

最简单的神经网络就是一个单神经元——做一个线性变换再加一个ReLU激活：

价格 = max(0, w × 面积 + b)

当输入特征变多（面积、卧室数、邮编、学区），中间加入多个隐藏神经元，就变成了标准神经网络。隐藏层自己学习哪些特征该组合在一起——你不需要手动告诉它"邮编+学区=地段质量"，它自己会发现。

不同数据用不同架构

数据类型 网络架构 应用举例
结构化数据（表格） 标准NN 房价预测、广告点击率
图像 CNN（卷积神经网络） 猫狗识别、自动驾驶
序列数据（文本/语音） RNN（循环神经网络） 机器翻译、语音识别
混合数据 混合架构 自动驾驶（雷达+图像）

💡 关键认知：深度学习在非结构化数据（图像、文本、语音）上的突破是近年来最大的进展。结构化数据上传统机器学习也能做得不错，但图像和语言这种人类天生擅长的事情，深度学习终于也能做了。

深度学习为什么现在才火？

三大推动力形成了完美风暴：

因素 具体表现
数据规模 互联网、手机、IoT产生了海量标注数据
计算能力 GPU、云计算让大模型训练成为可能
算法创新 ReLU替代Sigmoid解决了梯度消失，让训练速度大幅提升

规模效应：小数据集上，手工特征工程的SVM可能比神经网络好；但数据量足够大时，大模型+大数据的组合会持续碾压其他方法。

二、逻辑回归——最简单的神经网络

一句话：逻辑回归就是一个单神经元分类器——输入特征，输出"是猫的概率"。

猫图识别的问题设置

输入：一张64×64×3的彩色图片 → 展平为12,288维向量 x
输出：y = 1（是猫）或 y = 0（不是猫）
目标：学习 P(y=1|x)，即这张图是猫的概率

符号约定（贯穿整门课）：

m      = 训练样本数
nₓ     = 输入特征维度（这里是12288）
X      = (nₓ, m) 矩阵，m个样本按列堆叠
Y      = (1, m) 矩阵

📐 Sigmoid函数公式

Sigmoid函数

逻辑回归的核心是Sigmoid函数——把任意实数压缩到(0,1)之间：

                    1
σ(z) = ─────────────────
         1 + e⁻ᶻ
z值 σ(z) 含义
z → +∞ → 1 很确定是猫
z = 0 0.5 一半一半
z → -∞ → 0 很确定不是猫

Sigmoid的导数（后面反向传播要用）：

σ'(z) = σ(z) · (1 - σ(z)) = a(1-a)

最大值在z=0时取到，等于0.25。这个0.25就是Sigmoid导致梯度消失的根源——连乘50次就是10⁻⁶⁰。

📐 逻辑回归前向传播

完整前向传播

z = wᵀx + b          ← 线性变换
ŷ = σ(z)             ← 激活函数

损失函数——为什么不用均方误差？

单样本损失（交叉熵）：

L(ŷ, y) = -[y·log(ŷ) + (1-y)·log(1-ŷ)]

理解这个公式的直觉： - 当 y=1：L = -log(ŷ) → 希望ŷ尽可能大（接近1），否则损失很大 - 当 y=0：L = -log(1-ŷ) → 希望ŷ尽可能小（接近0），否则损失很大

整个训练集的成本函数（m个样本取平均）：

              1   m
J(w,b) = ─── Σ  L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾)
              m  i=1

💡 关键认知：为什么不用均方误差(MSE)？因为MSE+Sigmoid组合会导致成本函数非凸（有多个局部最小值）。交叉熵是凸函数，保证梯度下降能找到全局最优解。

三、梯度下降

一句话：梯度下降就是"沿着山坡最陡的方向往下走"，每走一步就更接近成本函数的最低点。

参数更新规则

w := w - α · ∂J/∂w
b := b - α · ∂J/∂b

其中 α 是学习率（步长大小），∂J/∂w 是成本函数关于w的偏导数（梯度方向）。

逻辑回归的梯度（m个样本）

         1   m
dw = ─── Σ  x⁽ⁱ⁾ · (ŷ⁽ⁱ⁾ - y⁽ⁱ⁾)
         m  i=1

         1   m
db = ─── Σ  (ŷ⁽ⁱ⁾ - y⁽ⁱ⁾)
         m  i=1

💡 关键认知：梯度的核心就是 (预测值 - 真实值) 乘以输入特征。形式非常简洁——预测越偏，梯度越大，参数更新越猛。

四、计算图与链式法则

一句话：计算图把复杂计算拆成一步步简单操作，正向算结果，反向算梯度。这就是反向传播的核心原理。

简单例子

计算 J = 3(a + bc)：

正向传播：
  b=3 ──×──→ u=6 ──┐
  c=2 ──┘          ├──(+)──→ v=11 ──×3──→ J=33
  a=5 ─────────────┘

反向传播（从右到左，用链式法则）：
  dJ/dv = 3
  dJ/da = dJ/dv × dv/da = 3 × 1 = 3
  dJ/du = dJ/dv × dv/du = 3 × 1 = 3
  dJ/db = dJ/du × du/db = 3 × c = 3 × 2 = 6
  dJ/dc = dJ/du × du/dc = 3 × b = 3 × 3 = 9

链式法则：每个节点的梯度 = 上游梯度 × 本地导数。就像接力赛一样，每一站接过前一站的"梯度棒"，乘上自己的"贡献因子"再传下去。

五、向量化——速度是for循环的300倍

一句话：永远不要用for循环遍历样本，用NumPy的矩阵运算替代——同样的计算，从200ms变成1.5ms。

非向量化 vs 向量化

# 非向量化（慢，200ms）
z = 0
for i in range(n_x):
    z += w[i] * x[i]
z += b

# 向量化（快，1.5ms）
z = np.dot(w, x) + b

m个样本的全向量化前向传播

Z = np.dot(W, X) + b          # (1, m) = (1, nₓ) × (nₓ, m)
A = sigmoid(Z)                 # 逐元素应用Sigmoid
dZ = A - Y                     # (1, m)
dW = (1/m) * np.dot(dZ, X.T)  # (1, nₓ)
db = (1/m) * np.sum(dZ)       # 标量

💡 关键认知：深度学习代码中几乎不应该出现显式for循环。所有操作都应该写成矩阵运算——这不仅是代码风格问题，而是300倍的速度差距。GPU/CPU的SIMD指令集天然支持矩阵并行。

Python广播（Broadcasting）

# (m, n) + (1, n) → 自动扩展为 (m, n)
# (m, n) + (m, 1) → 自动扩展为 (m, n)

重要提示：避免使用rank-1数组！

a = np.random.randn(5)      # ❌ shape: (5,) → 行为不可预测
a = np.random.randn(5, 1)   # ✅ shape: (5,1) → 列向量

六、浅层神经网络

一句话：在逻辑回归的基础上加一个隐藏层，模型就从"直线"变成了"曲线"——能拟合更复杂的边界。

网络结构

输入层 x → [隐藏层 a⁽¹⁾] → 输出层 ŷ
(nₓ,1)    (n₁, 1)           (1, 1)

前向传播公式

z⁽¹⁾ = W⁽¹⁾x + b⁽¹⁾       ← 第1层线性变换
a⁽¹⁾ = g⁽¹⁾(z⁽¹⁾)          ← 第1层激活
z⁽²⁾ = W⁽²⁾a⁽¹⁾ + b⁽²⁾    ← 第2层线性变换
ŷ = a⁽²⁾ = σ(z⁽²⁾)         ← 输出层Sigmoid

激活函数选择

激活函数 公式 范围 导数 用在哪
Sigmoid 1/(1+e⁻ᶻ) (0,1) a(1-a) 输出层（二分类）
tanh (eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ) (-1,1) 1-a² 隐藏层（比Sigmoid好）
ReLU max(0,z) [0,∞) 0或1 隐藏层首选
Leaky ReLU max(0.01z,z) (-∞,∞) 0.01或1 防止"死神经元"

💡 关键认知：为什么需要非线性激活？如果所有层都用线性激活（a=z），那么无论网络多深，整体效果等同于一个单层线性变换：a⁽²⁾ = (W⁽²⁾W⁽¹⁾)x + (W⁽²⁾b⁽¹⁾+b⁽²⁾) = W'x + b'。深度白费了！

随机初始化

W = np.random.randn(n_h, n_x) * 0.01    # ✅ 小随机数
b = np.zeros((n_h, 1))                    # ✅ 可以初始化为0

为什么不能全零初始化？ - 如果W全为0，所有神经元的输出完全相同 - 梯度也完全相同 → 更新也完全相同 - 整层神经元永远保持对称，相当于只有1个神经元 - 叫做"对称性问题"

七、深层神经网络

一句话：浅层网络只有1-2个隐藏层；深层网络有几十甚至上百层。深度的价值在于逐层抽象——底层学边缘，中层学部件，高层学整体。

为什么深度有用？

电路理论视角：某些函数用深度网络只需O(log n)个神经元，浅层网络却需要O(2ⁿ)个。

人脸识别的逐层抽象：

第1层：边缘检测（水平线、垂直线、对角线）
第2层：局部部件（眼睛、鼻子、嘴巴的形状）
第3层：人脸轮廓（把部件组合成完整的脸）

L层网络的前向传播

对于第l层（l = 1, 2, ..., L）：
  Z⁽ˡ⁾ = W⁽ˡ⁾A⁽ˡ⁻¹⁾ + b⁽ˡ⁾
  A⁽ˡ⁾ = g⁽ˡ⁾(Z⁽ˡ⁾)

其中 A⁽⁰⁾ = X（输入），A⁽ᴸ⁾ = ŷ（预测）

矩阵维度速查

W⁽ˡ⁾:  (n⁽ˡ⁾, n⁽ˡ⁻¹⁾)
b⁽ˡ⁾:  (n⁽ˡ⁾, 1)
Z⁽ˡ⁾:  (n⁽ˡ⁾, m)
A⁽ˡ⁾:  (n⁽ˡ⁾, m)

维度检查技巧：W的行数 = 当前层神经元数，W的列数 = 上一层神经元数。做乘法前先检查维度，能避免90%的bug。

反向传播（L层网络）

从输出层开始：
dZ⁽ᴸ⁾ = A⁽ᴸ⁾ - Y

对于每一层l（从L-1到1）：
dZ⁽ˡ⁾ = (W⁽ˡ⁺¹⁾ᵀ · dZ⁽ˡ⁺¹⁾) ⊙ g'⁽ˡ⁾(Z⁽ˡ⁾)

梯度计算：
dW⁽ˡ⁾ = (1/m) · dZ⁽ˡ⁾ · A⁽ˡ⁻¹⁾ᵀ
db⁽ˡ⁾ = (1/m) · sum(dZ⁽ˡ⁾, axis=1, keepdims=True)

参数更新：
W⁽ˡ⁾ := W⁽ˡ⁾ - α · dW⁽ˡ⁾
b⁽ˡ⁾ := b⁽ˡ⁾ - α · db⁽ˡ⁾

💡 关键认知：⊙ 是逐元素乘法（Hadamard积）。反向传播的核心就是链式法则的矩阵化实现——从输出层一路乘回输入层，每层乘上自己的"本地导数"。

参数 vs 超参数

参数（通过训练学习） 超参数（手动设置）
W⁽¹⁾, b⁽¹⁾, W⁽²⁾, b⁽²⁾, ... 学习率α
迭代次数
隐藏层数L
各层神经元数n⁽¹⁾,n⁽²⁾,...
激活函数选择
Mini-batch大小
正则化参数λ

第二课：改善深层神经网络

模型搭好了但效果不好——这门课教你怎么"调教"它。正则化防过拟合、优化器加速收敛、Batch Norm稳定训练，三板斧让深度网络从"能跑"变成"好用"。

八、偏差与方差——诊断模型的第一步

一句话：偏差(Bias)衡量"学得够不够"，方差(Variance)衡量"学得过不过"。这是调模型的第一诊断工具。

四种情况

情况 训练误差 验证误差 诊断 该做什么
高偏差 15% 16% 欠拟合 加大网络/训练更久
高方差 1% 11% 过拟合 更多数据/正则化
高偏差+高方差 15% 30% 又欠又过 两方面都要改
低偏差+低方差 0.5% 1% ✅ 理想 不用改

基准线（Bayes Error）：人类在这个任务上的错误率。如果人类错误率≈0%（如猫狗识别），训练误差15%就是高偏差；如果人类错误率≈15%（如模糊X光片），训练误差15%就不算高偏差。

吴恩达的"基本配方"

模型训练完 → 看训练误差
  ↓
训练误差高？（高偏差）
  → 加大网络 / 训练更久 / 换架构
  ↓
训练误差OK → 看验证误差
  ↓
验证误差高？（高方差）
  → 更多数据 / 正则化 / 换架构
  ↓
两个都低 → 完成！

💡 关键认知：深度学习时代有一个福利——加大网络几乎只减少偏差而不增加方差（前提是用正则化）。这意味着"Bigger is almost always better"，只要你有正则化兜底。

九、正则化——对抗过拟合的武器库

一句话：正则化就是给模型"加约束"——不让它记住训练数据的噪声，只学有用的规律。

L2正则化（权重衰减）

在成本函数后面加一项惩罚：

                  1   m                      λ
J(W,b) = ─── Σ  L(ŷ⁽ⁱ⁾, y⁽ⁱ⁾) + ─── ‖W‖²F
                  m  i=1                    2m

其中 ‖W‖²F 是Frobenius范数（所有权重的平方和），λ是正则化强度。

效果：λ越大 → 权重越小 → 网络越"简单" → 越不容易过拟合（但太大会欠拟合）。

Dropout正则化

随机关掉一部分神经元——每次训练迭代时，每个神经元有p的概率被暂时"清除"。

# Inverted Dropout实现
d = np.random.rand(a.shape[0], a.shape[1]) < keep_prob
a = a * d            # 随机清零部分神经元
a = a / keep_prob    # ← 关键！除以keep_prob来补偿

💡 关键认知：为什么要除以keep_prob？因为如果keep_prob=0.8，训练时只有80%的神经元参与，输出的期望值变成了原来的0.8倍。除以0.8后恢复原来的期望值，保证训练和测试时的输出规模一致。这就是"Inverted Dropout"。

方法 原理 适用场景

Early Stopping 验证误差开始上升时停止训练 通用，但不正交

十、优化算法——让梯度下降跑得更快

Mini-batch梯度下降

一句话：不用所有数据算一次梯度（太慢），也不用单个样本（太抖），用一小批（如64/128/256个样本）算一次。

方法 Batch大小 特点
Batch GD m（全部） 每步准确但很慢
Stochastic GD 1 每步很快但噪声大
Mini-batch GD 64-512 速度和稳定性的最佳平衡

指数加权平均

理解动量和Adam的基础：

vₜ = β · vₜ₋₁ + (1-β) · θₜ

β=0.9 相当于对最近 1/(1-0.9) = 10天的数据取平均。

课上用伦敦温度的例子： - β=0.9：平滑曲线，对应约10天平均 - β=0.98：更平滑但反应迟钝，约50天平均 - β=0.5：抖动太大，约2天平均

动量梯度下降（Momentum）

vdW = β · vdW + (1-β) · dW     ← 累积历史梯度方向
W := W - α · vdW

物理直觉：梯度是加速度，v是速度。球滚下山坡时，会在一个方向上持续加速，在垂直方向上抵消抖动。

RMSprop

sdW = β₂ · sdW + (1-β₂) · dW²    ← 累积梯度的平方
W := W - α · dW / √(sdW + ε)

效果：梯度大的方向被缩小（除以大的√s），梯度小的方向被放大。自动平衡不同参数的学习步长。

📐 Adam = Momentum + RMSprop

Adam优化器（Momentum + RMSprop）

vdW = β₁ · vdW + (1-β₁) · dW       ← 动量（一阶矩）
sdW = β₂ · sdW + (1-β₂) · dW²      ← RMSprop（二阶矩）
v̂dW = vdW / (1-β₁ᵗ)                 ← 偏差修正
ŝdW = sdW / (1-β₂ᵗ)                 ← 偏差修正
W := W - α · v̂dW / (√ŝdW + ε)

推荐超参数：β₁=0.9, β₂=0.999, ε=10⁻⁸

💡 关键认知：Adam = Adaptive Moment Estimation。它结合了Momentum（"记住方向"）和RMSprop（"自适应步长"）的优点。几乎所有深度学习项目的默认优化器——如果你不知道选什么，选Adam。

学习率衰减

训练后期应该"小步走"以更精确地逼近最优解：

α = α₀ / (1 + decay_rate × epoch)     # 最常用
α = α₀ × 0.95^epoch                    # 指数衰减
α = α₀ / √epoch                        # 根号衰减

十一、Batch Normalization——深度网络的"稳定器"

一句话：在每层的激活之前（或之后），对数据做归一化——让每层的输入分布稳定，训练更快更稳。

计算过程

对每个mini-batch：
  μ = (1/m) Σ zᵢ                   ← 计算均值
  σ² = (1/m) Σ (zᵢ - μ)²          ← 计算方差
  zᵢ_norm = (zᵢ - μ) / √(σ² + ε)  ← 归一化
  z̃ᵢ = γ · zᵢ_norm + β            ← 缩放和偏移（可学习参数）

γ和β是可训练参数——让模型自己决定要不要完全归一化。如果γ=√(σ²+ε), β=μ，就等于没做归一化。

为什么有效？

  1. 加速训练：每层输入分布稳定，可以用更大的学习率
  2. 减少对初始化的敏感性
  3. 轻微的正则化效果：mini-batch引入噪声，类似Dropout

💡 关键认知：Batch Norm解决的核心问题是Internal Covariate Shift——每层参数更新后，下一层看到的输入分布就变了，就像地基一直在晃，上面的楼很难建稳。BN让每层的"地基"稳住。

测试时的Batch Norm

测试时没有mini-batch，用训练过程中的指数加权平均估算μ和σ²。

一句话：逻辑回归只能做"是/否"的二分类，Softmax可以做"猫/狗/鸡/无"的多分类。

Softmax(zᵢ) = ─────────────

输出K个概率，加起来等于1。

第三课：结构化机器学习项目

这门课不讲数学，只讲决策方法论——面对一个ML项目，怎么高效地诊断问题、做出正确的技术决策、避免浪费时间。

十三、正交化——一次只调一个旋钮

一句话：好的ML系统设计应该让每个"旋钮"只影响一个方面——就像老式电视的旋钮，一个调亮度，一个调色彩，互不干扰。

四个独立目标： 1. 训练集表现好 → 更大网络/更好优化器 2. 验证集表现好 → 正则化/更多数据 3. 测试集表现好 → 更大验证集 4. 实际应用好 → 更换指标/调整数据分布

为什么Early Stopping不够"正交"：它同时影响训练集表现和过拟合程度——拧一个旋钮动了两个方面。吴恩达更推荐L2正则化，因为它只管过拟合，不影响你训练到收敛。

十四、评估指标——用数字说话

单一数字评估指标

核心原则：必须有一个单一数字来比较不同模型——否则你会在"A精度高但B召回高"的无尽纠结中浪费时间。

F1 = 2 × (Precision × Recall) / (Precision + Recall)
分类器 精确率 召回率 F1分数
A 95% 90% 92.4% ← 选这个
B 98% 85% 91.0%

满足指标 vs 优化指标

指标类型 作用 例子
优化指标（1个） 越高越好，用来排名 准确率
满足指标（N个） 达标就行，不用最优 运行时间<100ms

💡 关键认知：如果你有N个关心的指标，选1个作为优化指标，剩下N-1个作为满足指标。这样"选最好的模型"就变成了"在满足所有约束的前提下，选优化指标最高的"——决策变得清晰。

训练/验证/测试集划分

项目 划分原则
验证和测试必须同分布 都从你关心的实际应用数据中采样
训练集可以不同分布 可以用网上爬的+用户上传的混合
大数据时代 98%训练/1%验证/1%测试（百万级数据时1%就够了）

十六、迁移学习——站在巨人的肩膀上

十七、卷积基础——从边缘检测说起

一句话：卷积就是用一个小的"滤波器"在图片上滑动，每个位置算一个点积——不同的滤波器能检测不同的特征（垂直边缘、水平边缘、纹理...）。

边缘检测

用3×3的滤波器在6×6的图片上滑动，每次覆盖3×3区域，算点积得到一个数：

输入(6×6) * 滤波器(3×3) = 输出(4×4)

垂直边缘检测器：

 1  0 -1
 1  0 -1
 1  0 -1

左边亮右边暗 → 输出大正数（有垂直边缘）

填充（Padding）

类型 做法 输出尺寸 用途
Valid 不填充 n-f+1 不关心边缘
Same 填充p=(f-1)/2 n 保持尺寸不变

步长（Stride）

输出尺寸 = ⌊(n + 2p - f) / s⌋ + 1

n=输入尺寸, f=滤波器尺寸, p=填充, s=步长

三维卷积

彩色图片有3个通道（RGB），滤波器也是3维的：

输入: (6×6×3) * 滤波器: (3×3×3) = 输出: (4×4×1)
多个滤波器 → 多个输出通道 → 一层卷积完成！

架构 关键创新 层数 参数量

ResNet——残差连接（最重要的创新之一）

问题：理论上网络越深越好，但实际训练时超过20层后误差反而上升（退化问题）。

解决方案：跳跃连接（Skip Connection）

a⁽ˡ⁺²⁾ = g(z⁽ˡ⁺²⁾ + a⁽ˡ⁾)     ← 把输入直接加到2层后的输出上

💡 关键认知：为什么残差连接有效？因为如果额外的层学不到有用的东西，权重趋近于零，a⁽ˡ⁺²⁾≈g(a⁽ˡ⁾)=a⁽ˡ⁾，等于直接"跳过"这两层。网络至少不会变差——这就是为什么ResNet可以训练到152层。

MobileNet——深度可分离卷积

标准卷积 深度可分离卷积
一步到位：同时处理空间和通道 拆成两步：先空间(Depthwise)再通道(Pointwise)
计算量：f²×nᶜ×nᶜ'×n² 计算量：f²×nᶜ×n² + nᶜ×nᶜ'×n²
- 节省约10倍计算量

二十、人脸识别——Triplet Loss

一次学习（One-Shot Learning）

每个人只有1张照片，不能用传统分类器。解决思路：不学"这是谁"，而是学"两张脸像不像"。

孪生网络（Siamese Network）

图片A → CNN → 128维向量 f(A)
图片B → CNN → 128维向量 f(B)
距离 d = ‖f(A) - f(B)‖²
d < τ → 同一人
d ≥ τ → 不同人

Triplet Loss

每次训练用三元组（Anchor, Positive, Negative）：

L = max(0, ‖f(A)-f(P)‖² - ‖f(A)-f(N)‖² + α)

目标：让A和P的距离小于A和N的距离，至少差一个间隔α。

💡 关键认知：选"难的"三元组是关键——如果N本来就和A差很远，模型什么都学不到。要选那些长得像但不是同一人的Negative。

二十一、神经风格迁移

一句话：把一张照片的"内容"和一幅画的"风格"融合，生成一张既有照片内容又有绘画风格的新图。

成本函数

J(G) = α · J_content(C, G) + β · J_style(S, G)
  • 内容损失：用CNN中间层的激活值衡量，G和C在该层的特征表示应该相似
  • 风格损失：用Gram矩阵（通道之间的相关性），G和S的Gram矩阵应该相似
Gram矩阵：G[l]_kk' = Σᵢ Σⱼ a[l]_ijk · a[l]_ijk'

风格 = 不同通道之间的相关模式（如"橙色纹理"常和"粗笔触"一起出现）。

第五课：序列模型

从RNN的梯度消失困境出发，到LSTM/GRU的门控突破，再到Word2Vec词嵌入，最终抵达注意力机制和Transformer——这条路直接通向今天的GPT和Claude。

二十二、RNN——让网络有"记忆"

一句话：标准神经网络处理每个输入都是独立的——RNN让每一步的输出"记住"前面的信息，适合处理序列数据（文本、语音、时间序列）。

为什么需要RNN

问题 标准NN的困难
输入/输出长度不固定 网络结构必须预设固定大小
不同位置的特征不共享 "周二"在句首和句尾要分别学习
没有时序关系 第3个词不知道前2个词是什么

RNN前向传播

aₜ = g(Wₐₐ · aₜ₋₁ + Wₐₓ · xₜ + bₐ)    ← 隐藏状态更新
ŷₜ = g(Wᵧₐ · aₜ + bᵧ)                    ← 输出预测

关键：同一组参数 Wₐₐ, Wₐₓ, Wᵧₐ 在每个时间步共享——这就是RNN的参数效率。

RNN的多种架构

类型 输入→输出 应用举例
一对一 1→1 标准NN（不是RNN）
一对多 1→多 音乐生成
多对一 多→1 情感分析
多对多（等长） 多→多 命名实体识别
多对多（不等长） 多→多 机器翻译（Encoder-Decoder）

语言模型

RNN做语言模型的核心：预测下一个词的概率。

P("cats average 15 hours of sleep a day") = P(cats) × P(average|cats) × P(15|cats average) × ...

训练数据：大量文本。目标：给定前面的词，预测下一个词。这不就是今天GPT做的事吗？

二十三、LSTM与GRU——解决长期记忆

RNN的梯度消失问题

句子："The cat, which already ate ..., was full."

RNN很难让"was"记住很远处的"cat"（单数）——因为梯度在反向传播时会指数级衰减。

GRU（门控循环单元）

用更新门Γᵤ和重置门Γᵣ来控制"记住什么、忘记什么"：

Γᵤ = σ(Wᵤ[cₜ₋₁, xₜ] + bᵤ)         ← 更新门：要不要更新记忆？
Γᵣ = σ(Wᵣ[cₜ₋₁, xₜ] + bᵣ)         ← 重置门：要不要忽略旧记忆？
c̃ₜ = tanh(Wc[Γᵣ⊙cₜ₋₁, xₜ] + bc)   ← 候选记忆
cₜ = Γᵤ⊙c̃ₜ + (1-Γᵤ)⊙cₜ₋₁          ← 最终记忆：新旧混合

📐 LSTM三门控制

LSTM（长短期记忆）

比GRU更强大，三个门：

Γf = σ(Wf[aₜ₋₁, xₜ] + bf)    ← 遗忘门：忘掉什么？
Γᵢ = σ(Wᵢ[aₜ₋₁, xₜ] + bᵢ)   ← 输入门：记住什么？
Γₒ = σ(Wₒ[aₜ₋₁, xₜ] + bₒ)   ← 输出门：输出什么？

c̃ₜ = tanh(Wc[aₜ₋₁, xₜ] + bc)
cₜ = Γf⊙cₜ₋₁ + Γᵢ⊙c̃ₜ          ← 记忆更新
aₜ = Γₒ⊙tanh(cₜ)               ← 隐藏状态

💡 关键认知：LSTM的记忆更新公式 cₜ = Γf⊙cₜ₋₁ + Γᵢ⊙c̃ₜ 是关键——遗忘门控制保留多少旧记忆，输入门控制加入多少新信息。当遗忘门≈1、输入门≈0时，信息可以不衰减地传递任意远——这就解决了梯度消失。

对比 GRU LSTM
门数 2个 3个
参数量 较少 较多
效果 多数任务够用 极长序列更好
选择建议 默认先试GRU GRU不够时换LSTM

二十四、词嵌入——让机器理解语义

一句话：把每个词表示为一个高维向量（如300维），让语义相近的词在向量空间中距离相近。

为什么需要词嵌入

One-hot编码的问题：10000个词就是10000维，且任意两个词的距离都相等——"苹果"和"橙子"的距离 = "苹果"和"国王"的距离。这不合理。

Word2Vec

两种架构： - Skip-gram：给定中心词，预测上下文词 - CBOW：给定上下文词，预测中心词

训练后的词向量有类比推理能力：

e_king - e_man + e_woman ≈ e_queen

词嵌入的偏见问题

e_man - e_woman ≈ e_computer_programmer - e_homemaker

词嵌入会学到训练数据中的社会偏见。课上介绍了消除偏见的方法：找到"性别方向"→在该方向上中和化。

二十五、Seq2Seq与注意力机制

序列到序列模型

Encoder: "Jane visite l'Afrique" → 固定长度向量 c
Decoder: c → "Jane visits Africa"

问题：整个输入句子被压缩成一个固定向量——句子越长，信息丢失越多。

集束搜索（Beam Search）

不用贪心（每步选概率最高的词），而是保留B个最有可能的候选序列：

B=3时：每步保留3个最佳候选 → 最终选总概率最高的完整句子

注意力机制（Attention）

核心思想：翻译每个词时，不只看压缩后的c，而是回头看整个输入序列，给每个输入词分配一个"注意力权重"。

αₜ,ₜ' = softmax(eₜ,ₜ')         ← 当前输出对每个输入位置的关注程度
cₜ = Σₜ' αₜ,ₜ' · aₜ'           ← 加权求和得到上下文向量

💡 关键认知：注意力机制的本质是动态的、内容相关的检索——翻译"Africa"时，模型学会把注意力放在"l'Afrique"上。这个机制后来成为Transformer的核心，直接催生了GPT/BERT/Claude。

二十六、Transformer——一切的起点

一句话：抛弃RNN的顺序处理，完全用自注意力机制并行处理所有位置——更快、更强、能处理更长的序列。

自注意力（Self-Attention）

对输入序列中的每个词，计算三个向量： - Q（Query）：我在找什么？ - K（Key）：我是什么？ - V（Value）：我的信息是什么？

                    Q · Kᵀ
Attention(Q,K,V) = softmax(─────) · V
                     √dₖ

除以√dₖ的原因：防止点积值太大导致softmax梯度消失。

多头注意力（Multi-Head Attention）

不止一组QKV，而是h组（如h=8）
→ 每组从不同"视角"关注不同方面
→ 拼接后线性变换

类比：一组注意力看语法关系，一组看语义关系，一组看距离关系...多个视角互补。

Transformer完整架构

Encoder:
  输入 → 嵌入+位置编码 → [自注意力 → FFN] × N → 编码器输出

Decoder:
  输出(shifted) → 嵌入+位置编码 → [掩码自注意力 → 交叉注意力 → FFN] × N → 线性+Softmax → 预测

💡 关键认知：Transformer的突破不是"性能更好"，而是可以并行。RNN必须顺序处理（前一个算完才能算后一个），Transformer通过Self-Attention让所有位置同时互相关注，训练速度提升了几十倍。没有这个并行化，今天的万亿参数大模型根本训练不出来。
