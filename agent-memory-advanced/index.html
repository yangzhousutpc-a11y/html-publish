核心结论：Agent 记忆不是单一模块，而是一套借鉴操作系统虚拟内存、映射认知科学分类的"分层系统"。2025-2026 行业已经收敛到"工作记忆 + 情节记忆 + 语义记忆 + 程序记忆"的四层架构。

一、为什么要有记忆？

Zylos Research 2026 年 4 月的白皮书开宗明义地指出——

"Memory is the defining architectural challenge for production AI agents. A stateless LLM, however capable, cannot build relationships, learn from mistakes, or carry expertise across sessions without an explicit memory layer bolted on top."

一句话翻译：LLM 本身是"无状态函数"——给它 prompt、给你 response，转头就忘。想让 Agent 像个"老搭档"而不是"金鱼"，必须外挂记忆层。

典型痛点出现在 大约第 100 轮对话（Arun Baby 的实证）：token 爆炸、人设崩坏、前后矛盾。

二、四层记忆架构（生产环境共识）

CPU 寄存器① Working Memory 工作记忆

定义：当前会话里正在"过脑子"的原始消息、工具调用、临时状态

载体：LLM 的 context window 本身

容量：~2K-4K token（Letta 的 Core Memory 设计值）

生命周期：本轮对话结束即失效

内存+缓存② Episodic Memory 情节记忆

核心特征："temporal ordering — not just facts, but facts anchored to a moment in time"（事实 + 时间戳）

内容：对话日志、工具调用轨迹、交互序列

关键创新：Zep/Graphiti 的双时间注解——同时记录"事件发生时间"和"Agent 首次观察时间"，解决事实的时效性冲突

评估基准：LoCoMo（Snap Research，35 会话/300 轮/9000 token），当前最强模型 MAGMA 得分 0.7

长期硬盘③ Semantic Memory 语义记忆

核心特征："largely atemporal — what the agent believes to be currently true"（无时间的结构化事实）

内容：用户偏好、实体属性、领域知识、关系图谱

代表方案：

  • Mem0（48K GitHub stars，2025.10 融资 2400 万美元）：三阶段管道——提取 → 冲突检测 → 图更新，同时用向量+图
  • LangMem：LangGraph 官方方案，JSON 结构化存储
  • Cognee：六阶段 cognify 管道 + 自改进 memify
系统配置④ Procedural Memory 程序记忆

定义："how to do things"——学到的工作流、成功的工具调用序列、行为启发式

关键洞察："best represented as instructions, not data"——应该作为 system prompt 注入，不应该进向量库检索

典型载体：CLAUDE.md、AGENTS.md、.cursorrules——都是 Markdown 文件

2025 末突破：Claude Code 自动内存系统（v2.0.64+）——Claude 可在会话中自主往 CLAUDE.md 里写新条目

前沿方向：AutoDream（2026.2）——后台子 Agent 在 Agent 空闲时审查、整合内存文件，类比人类 REM 睡眠的"记忆巩固"

四、两个核心设计抉择

抉择 1：向量 vs 知识图（混合架构取胜）

Zylos 白皮书里一针见血的判断——

"Vectors encode similarity but not structure."
  • 纯向量：语义模糊召回强，但无法处理"谁管理批准 API 部署的那个人？"这种关系链遍历
  • 纯知识图：关系查询确定，但语义召回弱

2025-2026 生产共识：混合检索——向量找候选节点 → 图遍历拓展 → 重排送入 LLM。

⚠ 一个扎心数据：截至 2025 年，95% 的 RAG 系统仍只用向量，混合架构的改进空间巨大。

抉择 2：谁管理记忆（Agent 自治 vs 应用托管）

流派代表特点
Agent 自治 Letta（原 MemGPT） LLM 通过 function call 自己决定何时读写记忆（core_memory_append、archival_memory_search）。优点：应用零代码；缺点：token 成本高、延迟高
应用托管 Mem0、LangMem 应用主动 add()，后台异步提取结构化事实。优点：快、便宜；缺点：需要工程介入

生产环境的选型基本看"你愿意 Agent 多聪明、愿意付多少钱"。

这是 2025 年冒出来、2026 年变成一级告警的问题——

  • MINJA 攻击（Dong et al. 2025）：向 Agent 记忆注入恶意事实，95% 注入成功率、70% 攻击成功率
  • Echoleak 事件（2024）：邮件里隐藏 prompt，诱导 Agent 泄露用户隐私
  • A-MemGuard 发现：LLM 独立审查记忆条目时会漏掉 66% 的中毒内容——因为恶意内容单看都"像正常事实"
  • Palo Alto Networks Unit 42（2025）：把"通过 Agent 记忆的间接 prompt 注入"列为企业 AI 部署高置信度威胁
⚠ 简单说：记忆越久、越自主，攻击面越大。企业级落地必须配中毒检测 + 审计日志。

六、行业共识与选型建议

Zylos 白皮书的最终判断——

"No single storage paradigm dominates. Production-grade agents increasingly rely on hybrid architectures that layer both [vector + graph], with an LLM-managed interface that decides what to store, when to retrieve, and when to forget."
场景推荐方案理由
个人助手、跨会话记忆 Mem0 部署量最大、延迟最低、有商业背书
需要时间推理的场景 Zep/Graphiti 双时间知识图，时间问题准确率最高
需要 Agent 自主管理记忆 Letta OS 式分层，Agent 自己管分页
LangChain 生态 LangMem 开箱即用
代码类 Agent CLAUDE.md + 自动写入 程序记忆走 markdown 最稳
