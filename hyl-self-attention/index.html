这是一门把 Self-Attention 和 Transformer 讲得非常"接地气"的课程。相比同类教材动辄抛出 Attention(Q,K,V)=softmax(QKᵀ/√d)V 就开始推导，李宏毅老师的切入点更朴素——他先问一个问题：当神经网络的输入不再是一张固定大小的图，也不是一个定长向量，而是一组数量不定的向量时（比如一句话、一段语音、一张图、一个分子），我们该怎么办？ 这个问题的答案，就是 Self-Attention。

第一章：Self-Attention 是什么（P1）

1.1 从一个"怪问题"开始：输入是一组向量

传统的神经网络有个前提——输入大小固定。一张图就是 224×224×3，一个特征向量就是 100 维。但很多真实任务不符合这个前提：

  • 一句话：词数不定，每个词是一个 word embedding 向量（比如 300 维）。整句话就是一组向量。
  • 一段语音：每 10ms 取一帧特征（比如 80 维 MFCC），5 秒就是 500 帧，一组 500 个向量。
  • 一张图（看作 patch 集合）：把图切成 16×16 的 patch，每个 patch 一个向量。
  • 一个分子：每个原子一个向量，分子大小不同，向量数量不同。

这类输入叫 Vector Set（向量集合）。

图2｜Vector Set 输入场景：文字 / 语音 / 图 / 分子 都是「不定数量的向量集合」

1.2 输出长什么样

Vector Set 作为输入，输出可以有三种形态：

输出类型场景例子
N 对 N（每向量一个 label）Sequence Labeling词性标注 "I saw a saw" → [代词, 动词, 冠词, 名词]
N 对 1（整序列一个 label）序列分类情感分析、语音分类
N 对 M（seq2seq）序列到序列机器翻译、语音识别、对话

1.3 为什么 FC 层不够？

老师在这里抛出一个绝妙的反例："I saw a saw"（我看见一把锯子）。如果把每个词独立地送进 FC 做词性标注，两个 "saw" 的 word embedding 完全相同，FC 输出也必然相同——但第一个是动词、第二个是名词，就错了。

问题根源是：判断词性必须看 context。朴素解法是开个 window，但 window 再大也有限。Self-Attention 登场。

1.4 Self-Attention 的核心机制

Self-Attention 输入是一组向量 a¹, a², a³, a⁴，输出同样数量的向量 b¹, b², b³, b⁴，每一个 bⁱ 都是"考虑了整个序列之后"的结果。

图3｜Self-Attention 三步走：q/k/v 变换 → 点积+softmax 得权重 → 用 v 加权求和

第一步：计算关联度 α。对每个位置 j，计算 a¹ 和 aʲ 的关联度。最主流的 Dot-product Attention：

  • 把 a¹ 乘矩阵 Wq，得到 q¹ = Wq · a¹（query）
  • 把 aʲ 乘矩阵 Wk，得到 kʲ = Wk · aʲ（key）
  • 点积：α₁,ⱼ = q¹ · kʲ

1.5 矩阵形式：所有位置并行计算

Self-Attention 在 GPU 飞快运行的关键：一次矩阵乘法算完所有位置。

Q = Wq · I          (每列是 qⁱ)
K = Wk · I          (每列是 kⁱ)
V = Wv · I          (每列是 vⁱ)

A  = Kᵀ · Q         (关联度矩阵)
A' = softmax(A)     (按列做 softmax)
O  = V · A'         (输出矩阵)

可学习参数只有 Wq、Wk、Wv 三个矩阵。

💡 为什么用 Dot-product 而不是其他度量？

点积可以看作"余弦相似度"的未归一化版本——两个向量越接近平行，点积越大。这契合"关联度"的直觉：两个词在语义空间方向相近就应该互相关注。而且点积能彻底矩阵化，跑得特别快。

💡 Q/K/V 为什么要三套参数？

Query 代表"我在找什么"，Key 代表"我是什么（供别人检索）"，Value 代表"我能贡献什么信息"——三者语义上是不同角色。实验表明共享 Wq=Wk 会显著掉点。

1.6 名词解释

名词解释
Self-Attention一种让序列中每个元素都能"看到"整个序列其他元素并与之交互的机制。
Query（Q）当前位置发出的"查询"，用来去问其他位置"你跟我有多相关"。
Key（K）每个位置提供的"键"，用来被别的 Query 检索。
Value（V）每个位置实际提供的"值"，会被按关联度加权汇总。
Dot-product Attention用 Q 和 K 的点积计算关联度的 Attention 形式。
Sequence Labeling给序列中每个元素打一个标签，如词性标注、NER。

1.7 最小 PyTorch 实现

import torch
import torch.nn as nn
import torch.nn.functional as F

class SelfAttention(nn.Module):
    def __init__(self, d_model, d_k):
        super().__init__()
        self.Wq = nn.Linear(d_model, d_k, bias=False)
        self.Wk = nn.Linear(d_model, d_k, bias=False)
        self.Wv = nn.Linear(d_model, d_k, bias=False)
        self.d_k = d_k

    def forward(self, x):
        # x: [batch, seq_len, d_model]
        Q = self.Wq(x)
        K = self.Wk(x)
        V = self.Wv(x)
        scores = Q @ K.transpose(-2, -1) / (self.d_k ** 0.5)
        attn = F.softmax(scores, dim=-1)
        return attn @ V

第二章：Self-Attention 进阶（P2）

2.1 Multi-Head Self-Attention（多头自注意力）

单个 Self-Attention 只学到一种"关联模式"，但两个词之间的关联可能有多种类型——语法、语义、位置、指代。多头的做法是：并行学多套 Q/K/V，最后拼起来。

图5｜Multi-Head：不同头学习不同关联模式（语法/语义/指代），最后拼接融合

如 d_model = 512、h = 8，每个头 d_k = d_v = 64。多头不增加参数量，只是把维度"切片"让不同片关注不同模式。

2.2 Positional Encoding（位置编码）

Self-Attention 有个"隐藏病"：对输入顺序不敏感（permutation invariant）。打乱 "猫追狗" 三个词的顺序，模型分不清它和 "狗追猫"。于是给每个位置加 eⁱ：

aⁱ' = aⁱ + eⁱ

Transformer 原论文用 sin/cos：

好处：每个位置唯一编码、相对位置关系可线性表达、可外推到更长序列。也可以用可学习 position embedding（BERT）或相对位置编码（T5、RoPE）。

💡 位置编码为什么是"相加"而不是"拼接"？

拼接会让输入维度变大、增加参数；相加把位置信息"植入"到每个维度里，实验表明效果相当。底层数学上，相加等价于在更高维空间里一种特殊的拼接——位置编码维度和 embedding 维度被强制共享一个空间。

2.3 Self-Attention 应用场景

领域应用
NLPTransformer、BERT、GPT——几乎所有现代语言模型
语音Truncated Self-Attention：语音序列几千帧，N² 扛不住，只看附近 window
图像切 patch 当 vector 集合 → ViT 的雏形
生成Self-Attention GAN

2.4 Self-Attention vs CNN

论文《On the Relationship between Self-Attention and Convolutional Layers》证明：CNN 是 Self-Attention 的特例。CNN 感受野是人为设定，Self-Attention 的"感受野"是学出来的。

  • 数据少时，CNN 更好（平移不变性是强 inductive bias）
  • 数据多时，Self-Attention 更好（ViT：训练数据超 JFT-300M 量级时反超 CNN）
图6｜CNN vs Self-Attention：感受野固定 vs 学出来；数据少用 CNN，多用 SA

2.5 Self-Attention vs RNN

对比项RNNSelf-Attention
并行性串行完全并行

2.6 名词解释

名词解释
Multi-Head Attention并行多组 Q/K/V 学习不同类型关联，最后拼接融合。
Positional Encoding给每个位置加"身份向量"，让 Self-Attention 感知顺序。
Truncated Self-Attention只对附近 window 做 attention，用于超长序列。

2.7 Multi-Head Attention 实现

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, h):
        super().__init__()
        assert d_model % h == 0
        self.h = h
        self.d_k = d_model // h
        self.Wq = nn.Linear(d_model, d_model)
        self.Wk = nn.Linear(d_model, d_model)
        self.Wv = nn.Linear(d_model, d_model)
        self.Wo = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, N, _ = x.shape
        Q = self.Wq(x).view(B, N, self.h, self.d_k).transpose(1, 2)
        K = self.Wk(x).view(B, N, self.h, self.d_k).transpose(1, 2)
        V = self.Wv(x).view(B, N, self.h, self.d_k).transpose(1, 2)

        scores = Q @ K.transpose(-2, -1) / (self.d_k ** 0.5)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn = F.softmax(scores, dim=-1)
        out = attn @ V

        out = out.transpose(1, 2).contiguous().view(B, N, -1)
        return self.Wo(out)

第三章：Transformer Encoder（P3）

3.1 Seq2Seq：万物皆可序列

Transformer 是一个 Seq2Seq 模型——输入序列，输出序列，输出长度由机器自己决定。应用广度惊人：

  • 机器翻译、语音识别、语音合成（TTS）
  • 对话系统、语法分析（树序列化）
  • 多标签分类、目标检测（DETR）

经典架构是 Encoder-Decoder：Encoder 理解输入，Decoder 一个 token 一个 token 生成输出。

图7｜Transformer 整体架构：Encoder 理解输入，Decoder 逐 token 生成

3.2 Encoder 整体结构

Encoder 由 N 个相同 Block 堆叠（原论文 N=6）。每个 Block 包含两个子层，每个子层后接 Add + LayerNorm。

图8｜Encoder Block 内部：SA → Add&LN → FFN → Add&LN，两次残差跨越子层

3.3 残差连接（Residual Connection）

x' = x + Sublayer(x)

思想来自 ResNet：让网络先输出"改动量"，而不是直接输出新值。如果 Sublayer 一开始学不到东西，x' ≈ x，信息不丢失；梯度能顺畅回传到浅层。

3.4 Layer Normalization

Transformer 用 Layer Norm，不是 Batch Norm：

归一化归一化维度适用
Batch Norm对 batch 内每个特征通道分别归一化CNN、图像
Layer Norm对单样本所有特征一起归一化NLP、序列

为什么 NLP 用 LN？序列长度不一、BN 在变长样本上统计量不稳定；同位置不同句子语义差异极大，BN 反而抹掉这种差异；padding 会污染 BN 统计量。

3.5 FeedForward Network

若 d_model = 512，FFN 中间层 d_ff = 2048（先扩 4 倍再缩回）。FFN 是"对每个位置独立做非线性变换"，补充 Self-Attention 的线性融合能力。

3.6 完整 Block 的数学表达

Post-LN（原版）：

x' = LayerNorm( x + MultiHead(x) ) out = LayerNorm( x' + FFN(x') )

Pre-LN（后来更流行）——训练更稳，GPT-3、LLaMA 都用：

x' = x + MultiHead( LayerNorm(x) ) out = x' + FFN( LayerNorm(x') )

3.7 名词解释

名词解释
Seq2Seq输入输出都是序列、输出长度由模型决定的架构范式。
Encoder把输入序列映射为一组带上下文的语义向量。
Residual Connectionoutput = Layer(x) + x，让梯度顺畅回传。
Layer Normalization单个样本内对所有特征归一化，适合 NLP。
FeedForward Network两层 MLP，对每个位置独立做非线性变换。
Post-LN / Pre-LNLN 放在残差之后/之前的两种变体。

3.8 Encoder Block 实现

class EncoderBlock(nn.Module):
    def __init__(self, d_model, h, d_ff, dropout=0.1):
        super().__init__()
        self.attn = MultiHeadAttention(d_model, h)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.ReLU(),
            nn.Linear(d_ff, d_model),
        )
        self.ln1 = nn.LayerNorm(d_model)
        self.ln2 = nn.LayerNorm(d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, x, mask=None):
        # Post-LN 原版
        x = self.ln1(x + self.dropout(self.attn(x, mask)))
        x = self.ln2(x + self.dropout(self.ffn(x)))
        return x
💡 残差连接 + LN 为什么如此关键？

残差保证梯度有"高速公路"反传到浅层；LayerNorm 保证每层输入数值范围稳定。两者缺一不可——去掉残差深层训不起来；去掉 LN 数值会漂移。

第四章：Transformer Decoder（P4）

4.1 Decoder 的两种范式：AT 与 NAT

Autoregressive（AT） 主流做法：

  • 初始喂特殊 token BEGIN
  • 输出第一个 token 的概率分布，得到 y₁
  • 把 y₁ 作为输入，得到 y₂
  • 循环直到输出 END

Non-Autoregressive（NAT） 则并行一次吐出整个序列，速度快但质量通常不如 AT，存在 multi-modality problem（翻译 "thank you" 可能是"谢谢"也可能是"感谢"，NAT 会把两种混在一起输出怪话）。

4.2 Decoder 内部结构

Decoder Block 比 Encoder 多一层——三层子结构：Masked Self-Attention → Cross-Attention → FFN。

图9｜Decoder Block：Masked-SA → Cross-Attention → FFN 三层结构

4.3 Masked Self-Attention

为什么要 mask？ Decoder 预测第 t 个 token 时，只能看到前 t-1 个。如果允许 Self-Attention 看到未来位置，训练时模型直接"偷看答案"，测试时无答案可看，训练测试就对不上了。

怎么 mask？ 在 softmax 之前，把"未来位置"的 score 设为 -∞：

scores = Q Kᵀ / √d_k                    # shape [N, N]
scores[i, j] = -∞   for all j > i        # 上三角置 -∞
attn = softmax(scores)
out  = attn V
图10｜Masked Attention：上三角置 -∞ → softmax 后严格为 0 → Decoder 只看到过去
💡 为什么用 -∞ 而不是 0？

softmax 前如果置 0，softmax 后仍有非零权重（exp(0)=1）。必须用 -∞（工程实现 -1e9），softmax 后才能严格为 0。

4.4 Cross-Attention：Decoder 如何向 Encoder 提问

Cross-Attention 是 Encoder-Decoder 的"信息桥梁"：

  • Q 来自 Decoder 当前位置的隐状态
  • K、V 来自 Encoder 的输出

直觉：Decoder 生成到一半，想知道"接下来应该关注 Encoder 输入的哪些部分"，用自己的 Q 和 Encoder 所有 K 算关联度，从 Encoder 的 V 加权取回信息。

图11｜Cross-Attention：Decoder 用自己的 Q 去检索 Encoder 的 K/V，实现「理解→生成」的连接

原始论文中，Decoder 每个 Block 都用 Encoder 最后一层输出做 Cross-Attention。

4.5 训练：Teacher Forcing

训练 Decoder 时用 Teacher Forcing：每步不喂模型上一次预测，直接喂 ground truth。以 "我爱猫 → I love cats" 为例：

  • Step 1：输入 BEGIN，目标输出 "I"
  • Step 2：输入 BEGIN + "I"（ground truth，不管模型上步是否输出 "I"），目标 "love"
  • Step 3：输入 BEGIN + "I" + "love"，目标 "cats"
  • Step 4：输入 BEGIN + "I" + "love" + "cats"，目标 END

损失函数就是每个位置的 Cross-Entropy 求和。

💡 Teacher Forcing 的代价：Exposure Bias

训练时模型从未见过自己犯错，每步输入都是完美 ground truth。预测时一旦某步错了，后面就基于"错误的上文"继续生成，错误会连锁放大（走上训练时从未走过的路）。缓解方法是 Scheduled Sampling：训练时以一定概率用模型自己输出代替 ground truth。

4.6 预测：Beam Search

最朴素是 Greedy（每步选概率最大的 token），但 Greedy 是局部最优。

Beam Search 保留 top-K 条候选路径（beam width = K）：每步对每条路径扩展所有可能，再留 K 条累计概率最高的，直到所有路径都碰到 END。

  • 对答案唯一任务（翻译、摘要）：Beam Search 稳定更好
  • 对答案开放任务（生成故事）：Beam Search 反而枯燥重复，随机采样 / Top-K / Top-p 更好

4.7 其他训练技巧

  • Copy Mechanism：让 Decoder 从 Encoder 输入复制 token，用于聊天机器人、摘要（处理人名、专名）。代表：Pointer Network、CopyNet。
  • Guided Attention：强制 attention 按某种顺序（语音合成必须从左到右），加正则约束 attention 矩阵只出现在对角线附近。
  • Optimizing Evaluation Metrics：训练用 CE、评测用 BLEU，两者不一致。BLEU 不可微，用强化学习（REINFORCE）绕过。

4.8 名词解释

名词解释
Autoregressive（AT）逐 token 生成，前一步输出作为下一步输入。
Non-Autoregressive（NAT）一次并行生成整个序列，速度快但质量稍差。
Masked Self-Attention把未来位置 score 设为 -∞，让 Decoder 只能看到过去。

Beam Search保留 top-K 候选路径的启发式搜索。
Copy Mechanism允许 Decoder 从输入复制 token。
Scheduled Sampling训练时有概率用模型预测作为输入，缓解 Exposure Bias。

4.9 Masked Attention 片段

def masked_attention(Q, K, V, d_k):
    N = Q.size(-2)
    scores = Q @ K.transpose(-2, -1) / (d_k ** 0.5)
    # 下三角 mask（True 表示可见）
    mask = torch.tril(torch.ones(N, N, dtype=torch.bool, device=Q.device))
    scores = scores.masked_fill(~mask, float('-inf'))
    attn = F.softmax(scores, dim=-1)
    return attn @ V

第五章：Self-Attention 变型（P5）

5.1 为什么需要变型？N² 的诅咒

普通 Self-Attention 计算量 O(N² · d)——N 越大，attention matrix 爆炸：

N（序列长度）Attention Matrix 大小显存压力
512（BERT）26 万元素勉强可跑
4096（长文档）1680 万吃紧
16384（基因/图像 Patch）2.68 亿单卡放不下
图12｜Efficient Transformer 五大路线：稀疏 / LSH / 低秩 / 核近似 / 质疑

5.2 思路一：Local / Sparse Attention

  • Local Attention（Longformer）：只看附近 window（128~512）
  • Stride Attention：跳着看（第 1、3、5...）
  • Global Attention：少数特殊 token（如 [CLS]）看全部
  • Big Bird：Local + Random + Global 三合一
  • Data-dependent Sparse（Routing Transformer）：K-means 聚类 Q/K，只在同簇内算

5.3 思路二：Reformer 的 LSH 分桶

核心观察：Attention 里真正重要的是 Q、K 点积最大的那些对。LSH（局部敏感哈希）能把相似向量以高概率哈希到同一个桶，只在同桶内算 attention。

h(x) = sign( R · x ) (R 是随机矩阵)

复杂度降到 O(N log N)。

5.4 思路三：Linformer 的低秩降维

观察：Attention 矩阵在真实数据上近似低秩——不需要 N×N，只需 k 个主要维度（k ≪ N）。把 K、V 沿序列维度投影到 k 维：

K' = E · K ∈ R^(k × d) V' = F · V ∈ R^(k × d)

attention matrix 从 N×N 降为 N×k，复杂度 O(N · k · d)，k 为常数时等效线性。

5.5 思路四：Linear Attention / Performer 的线性化

这是最优雅的一类思路。问题本质：能不能绕过 softmax，直接把矩阵乘法结合律用起来？

图13｜线性 Attention 的核心：绕过 softmax 用矩阵结合律，从 O(N²) 降到 O(N·d²)

原版必须先算 QKᵀ（N×N）才能做 softmax——N² 的来源：

如果没有 softmax，可以用结合律重排：

Linear Transformer 用 φ(x) = elu(x) + 1 代替 softmax；Performer 更进一步用随机特征近似 softmax 核：

softmax(q · k) ≈ φ(q) · φ(k)

5.6 思路五：Synthesizer 的大胆质疑

Synthesizer 直接问：attention matrix 真的有必要由 Q·Kᵀ 生成吗？ 实验表明：

  • 让 attention matrix 直接作为可学习参数，性能下降有限
  • 甚至用随机矩阵作为 attention，很多任务也能跑得不错

5.7 Long Range Arena（LRA）的启示

Google 推出的 LRA 基准结论：

  • 几乎所有变型都比原版快、内存小
  • 但准确率上真正能稳定超过原版的并不多
  • 没有免费午餐——把 N² 压成线性的代价通常是某种能力损失

工业界到今天，对中短序列（N ≤ 几千）依然首选原版 Transformer；只有真正超长序列（基因、长文档、视频）才挑特定高效变型。

💡 线性 Attention 的代价是什么？

把 softmax 换成核近似 φ，表达能力会缩水——原 softmax 是非线性的、能自适应分配注意力；φ 是固定映射，加上精度损失，在复杂任务上容易掉点。这就是为什么 LRA 上大多数变型赢在速度、输在精度。

5.8 名词解释

名词解释
Local/Sparse Attention每位置只在局部 window 或稀疏模式内算 attention。代表 Longformer、Big Bird。
LSH局部敏感哈希，把相似向量哈希到同一桶。Reformer 用此实现 O(N log N)。
Linformer把 K/V 沿序列维度投影到低维 k，复杂度 O(N·k)。
Linear Attention / Performer用核近似 softmax，利用矩阵结合律把复杂度降到 O(N·d²)。
Synthesizerattention matrix 作为可学习参数或随机矩阵，挑战 Q·Kᵀ 必要性。
Long Range Arena（LRA）Google 推出的长序列任务基准。

5.9 Linear Attention 简化实现

class LinearAttention(nn.Module):
    """用 elu(x)+1 作为特征映射（Katharopoulos et al., 2020）"""
    def __init__(self, d_model, h):
        super().__init__()
        self.h = h
        self.d_k = d_model // h
        self.Wq = nn.Linear(d_model, d_model)
        self.Wk = nn.Linear(d_model, d_model)
        self.Wv = nn.Linear(d_model, d_model)
        self.Wo = nn.Linear(d_model, d_model)

    def feature_map(self, x):
        return F.elu(x) + 1

    def forward(self, x):
        B, N, _ = x.shape
        Q = self.Wq(x).view(B, N, self.h, self.d_k).transpose(1, 2)
        K = self.Wk(x).view(B, N, self.h, self.d_k).transpose(1, 2)
        V = self.Wv(x).view(B, N, self.h, self.d_k).transpose(1, 2)

        Q, K = self.feature_map(Q), self.feature_map(K)

        # 关键：先算 KᵀV（d×d），再左乘 Q（N×d）
        KV = K.transpose(-2, -1) @ V
        Z  = 1 / (Q @ K.sum(dim=-2, keepdim=True).transpose(-2, -1) + 1e-6)
        out = (Q @ KV) * Z

        out = out.transpose(1, 2).contiguous().view(B, N, -1)
        return self.Wo(out)

第六章：串起整个课程

6.1 四小时到底讲了什么？

从"处理向量集合"这个朴素问题出发，李老师一路讲到最基础的 Self-Attention（Q/K/V、Dot-product、加权和）→ Multi-Head → Positional Encoding → Encoder-Decoder 架构 → Decoder 的 Masked + Cross Attention → 训练 Teacher Forcing、预测 Beam Search、Exposure Bias → 最后面对 N² 瓶颈的各种 Efficient 变型。

整个讲解主线是"为什么"而不是"是什么"：为什么要 Q/K/V 三套？为什么要多头？为什么要位置编码？为什么 NLP 用 LN 不用 BN？为什么 Masked 用 -∞？这种讲法让知识变成理解，而不是记忆。
